## Binding contracts

- **Architecture / layering:** Product layer: products/<slug>/ — self-contained (own README, tests, runnable artifact, own pinned deps), NO cross-product imports. Coordination layer: control/ (inbox=manager-written ORDERs, status=coordinator heartbeat). Contract layer: README.md + CONVENTIONS.md + docs/. Repo-level tooling stays stdlib-only.
- **Ownership** (who owns each write path): Sole-writer per file: control/inbox.md manager-only; control/status.md coordinator-only. Each product subtree is claimed (claims/, one file per claim) while worked; parallel workers claim disjoint products.
- **Mutation seam** (how writes are gated): Every change lands via a PR merged on green (kit ceremony, born-red card, READY never draft, land-your-own per R21). Money/external-account steps are NEVER executed — they become conservative six-field owner plans (Q-0259 r.4). Releases via the proven workflow_dispatch recipe.
