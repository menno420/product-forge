## Ownership model

Sole-writer per file: control/inbox.md manager-only; control/status.md coordinator-only. Each product subtree is claimed (claims/, one file per claim) while worked; parallel workers claim disjoint products.

## New areas

A new products/<slug>/ subtree is created ONLY by a routed inbox ORDER (the manager applies the no-owning-lane test) — the forge never invents product intent. A product that outgrows its subtree graduates to a dedicated repo (owner click), becoming a lane.
